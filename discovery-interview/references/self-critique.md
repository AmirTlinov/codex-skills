# Self-critique (обязательный блок)

Перед возвратом всегда добавляй короткий блок с 2–3 тезисами:

1. Что могло быть упрощено.
2. Где вероятны недосказанности.
3. Что нужно проверить ещё раз у владельца задачи.

## Шаблон

```text
<self-critique>
- [ ] Я мог(ла) слишком рано зафиксировать путь без уточнения альтернатив.
- [ ] Есть риск, что 1 из важных ограничений не был явно подтверждён пользователем.
- [ ] Следующим шагом лучше попросить подтверждение по [критичному риску / приоритету / дедлайну].
</self-critique>
```

- Не более 4 пунктов.
- Формулировки конструктивные, без самокритики ради критики.
